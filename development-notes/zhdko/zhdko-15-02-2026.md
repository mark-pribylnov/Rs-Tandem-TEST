**Описание**  
Первым компонентом для либы выбрал кнопку, так как она точно всем понадобится первой.
Столкнулся сразу со множеством проблем, в основе которых: абсолютное незнание Angular, поэтому первые несколько часов потратил на чтение документации.  
Каждый раз, встречаясь с новым термином, уходил гуглить по разным ресурсам в попытках понять, как все устроено изнутри, но понятнее не становилось, поэтому спрашивал у ИИ. Понятнее все еще не становилось, так как сложно перескочить с нативного TS, где ты описывал все самостоятельно, а тут за тебя уже все сделано. Главное: понял основу, что в основе лежат сигналы, которые отвечают за состояние, рендер, принятие аргументов, ну почти за все.  
В процессе разработки захотел добавить кнопке вариант с иконкой, поэтому добавил пакет angular-svg-icon для более удобной работы с ними.

**Проблемы**

1. Не понимал, как передать клик из кнопки наружу родителю и прокинуть объект события, для предотвразения всплытия клика.
2. Не понимал, как следить за изменением состояния кнопки (например, когда она блокируется через `isDisabled`). В обычном JS мы бы вешали слушатель, а в Angular нужно было понять, как заставить компонент мгновенно перерисовываться, когда родитель меняет настройки.

**Решение**

1. Использовал механизм `output()` и метод `emit()`.
   Внутри компонента я перехватываю нативный клик, провожу проверку на `isDisabled()` и, если всё ок, вызываю `this.clicked.emit(event)`
   <details>
   <summary>код</summary>

   ```typescript
   readonly clicked = output<MouseEvent>();

   onClick(event: MouseEvent): void {
       if (!this.isDisabled()) {
         this.clicked.emit(event);
       }
   }
   ```

  </details>

2. Использовал сигналы `computed`. Вместо того чтобы вручную проверять изменения, я создал зависимую переменную:

```typescript
 readonly isDisabled = computed(() => this.config().isDisabled)
```

Теперь Angular сам следит за объектом конфига.
