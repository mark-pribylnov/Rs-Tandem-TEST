# Дата: 20.02.2026

**Описание**  
Следующим компонентом для библиотеки выбрал чекбокс. После опыта с инпутом реализация прошла гораздо быстрее. Основной упор сделал на кастомную стилизацию. Чтобы все работало четко, обернул структуру в тег label. Это позволило сделать всю область кликабельной без лишних манипуляций с id и for. Также решил начать составлять подробную документацию для каждого компонента, чтобы членам команды было проще ориентироваться в библиотеке.

**Проблемы**

1. Вначале по привычке пытался использовать строковые значения для состояния, но быстро понял, что для чекбокса логичнее работать со строгим boolean.
2. Столкнулся с тем, что при клике на текст галочка иногда не срабатывала из-за неправильной вложенности элементов.

**Решение**

1. Перевел всю логику ControlValueAccessor на работу с true/false. В методе writeValue добавил принудительное приведение к булеву типу через двойное отрицание, чтобы избежать ошибок, если из формы прилетит null.
2. Обернул весь компонент в общую label.

**План**  
В планах внедрить Storybook. Это позволит визуально тестировать компоненты в разных состояниях и сделает документацию еще нагляднее для всей команды.
